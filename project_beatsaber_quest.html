Beat Saber Modding (Quest)

About Beat Saber

Beat Saber is a music/rythm game where you slash blocks with light-sabers to the beat of a song. Modding is a big part of the game and it would not be the same without it. With mods you can add your own music to the game, customize the game looks, etc.

Beat saber is made in Unity game engine, where the C# is used as a scripting language.

Oculus Quest

Oculus Quest 2 is a standalone VR headset that uses Snapdragon XR2 processor. It is a powerful SoC with CPU 6x 1.8GHz and 2x 2.5GHz cores and GPU at 587MHz with up to 1267 GFlops performance. However, due to lack of efficient cooling the cpu cores are limited to 1.5GHz and 1.9GHz.

Beat Saber Oculus Quest

The game itself is built on Unity game engine using C# but due to hardware limitation the Quest version of the game is compiled with IL2CPP(Intermediate Language To C++) meaning that the mods "must" be created with C++. The modding tools are created and maintained by modders community. Basic principle behind modding is that function calls are interrupted by your custom code, either before or after the original function call.

Finger Saber

Latest mod

In this mod I use Oculus Quest hand-tracking API to set the position of the saber to the tips of your fingers. The game world is shrinked down to accommodate for smaller saber movement.

Biggest issue that I faced was memory leak that I could not explain. This is to be expected when making unofficial mods for a game where the tools are not very mature nor well documented. The memory leak issue was caused by hand-tracking container. This container was simply an Unity gameObject that contained all neccecary gameObjects and scripts. In Unity you can set a gameObjects to persist through the changes of scenes. For some reason hand-tracking container did no properly persist through the scene changes. My solution to this was to delete the hand-tracking container before scene changed and re-create it when a scene is loaded.

Flail Saber

Nunchucks / flails mod

In this mod the sabers are attached to rope. The rope is simulated using Verlet integration, and it is highly customizable. By changing parameters its possible to make rope more elastic or more rigid, longer or shorter, etc.

In Verlet integration the rope is simulated by moving the rope nodes by changing their position with respect to the forces acting on them. At this point the nodes "do not see" other nodes. After this, constrainted are applied multiple times (10-100 times). Each time rope points are moved slighlty to fit their constraints, i.e., the rope is "stretched" or "compressed" to fit the length constraint.
